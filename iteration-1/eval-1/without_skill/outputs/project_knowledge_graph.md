# 视频脚本生成 Agent 项目知识图谱

## 1. 技术栈与架构模式

### 核心技术栈
- **后端框架**: NestJS
- **Agent 编排**: LangGraph.js
- **消息队列**: BullMQ (基于 Redis)
- **数据库**: PostgreSQL + Prisma ORM
- **缓存**: Redis
- **LLM 集成**: OpenAI / Claude (待定)
- **前端通信**: SSE (Server-Sent Events) 流式推送

### 架构模式
- **前后端分离**: NestJS 后端 + 前端独立应用
- **异步任务处理**: BullMQ 消息队列解耦长任务
- **状态机编排**: LangGraph.js 管理 Agent 执行流程
- **模块化设计**: NestJS Module 按功能域划分

## 2. 关键架构决策清单

### 决策 1: 选择 LangGraph.js 而非 LangChain LCEL 进行 Agent 编排
- **决策描述**: 使用 LangGraph.js 的 StateGraph 管理视频脚本生成的多步骤状态流转
- **背景**: 脚本生成需要多步骤状态管理、条件分支、可中断/可恢复执行
- **取舍**:
  - 方案 A (LangChain LCEL): 只支持线性/有限分支的固定流程，无法表达循环
  - 方案 B (LangGraph.js): 支持状态图描述复杂 Agent 逻辑，包含循环和条件路由
- **最终选择**: LangGraph.js，因为需要状态管理和复杂控制流
- **置信度**: 🟡 中 (session 有明确讨论，但代码实现未确认)

### 决策 2: 使用 BullMQ 而非直接并发调用 LLM
- **决策描述**: 通过 BullMQ 队列管理批量脚本生成任务的并发和重试
- **背景**: 批量生成多个脚本时，直接并发调用 LLM 有 API 限流风险
- **取舍**:
  - 方案 A: 全部并发，速度快但 API 限流风险高
  - 方案 B: 队列串行，稳定但慢
  - 方案 C: 限速并发 (concurrency limit)，平衡速度和稳定性
- **最终选择**: BullMQ 实现限速并发，支持重试和状态追踪
- **置信度**: 🟡 中 (session 提及，代码结构推断)

### 决策 3: NestJS 而非 Express/Koa/Fastify
- **决策描述**: 使用 NestJS 作为后端框架构建视频脚本生成平台
- **背景**: 需要依赖注入、模块化、TypeScript 原生支持的中大型应用架构
- **取舍**:
  - 方案 A (Express): 灵活但缺乏架构约束，大型项目易混乱
  - 方案 B (NestJS): 提供完整架构模式，适合中大型应用
- **最终选择**: NestJS，因为需要模块化和依赖注入支持
- **置信度**: 🟡 中 (session 提及，但具体理由未详细记录)

### 决策 4: Streaming vs 批量返回输出
- **决策描述**: 实现流式输出 (SSE) 而非等待全部生成完再返回
- **背景**: 脚本生成较慢，用户等待体验差
- **取舍**:
  - 方案 A (Streaming): 实时流式输出，用户体验好；实现复杂，SSE/WebSocket 需要额外处理
  - 方案 B (批量返回): 实现简单，但用户体验差
- **最终选择**: Streaming 输出，改善用户体验
- **置信度**: 🟡 中 (session 有讨论，实现细节未确认)

### 决策 5: 状态持久化与检查点设计
- **决策描述**: 使用 LangGraph.js 的 Checkpointer 实现状态持久化和断点续传
- **背景**: 批量任务需要支持中断后恢复执行
- **取舍**:
  - 方案 A (MemorySaver): 内存持久化，适合开发调试
  - 方案 B (PostgresSaver): PostgreSQL 持久化，适合生产环境
- **最终选择**: PostgreSQL Checkpointer，支持生产级持久化
- **置信度**: 🟡 中 (从 LangGraph.js 学习材料推断)

## 5. 项目业务理解

### 业务痛点
- 公司业务同学和市场部同学经常有外投 DSP 视频的需求
- 当前视频脚本完全依赖手写，效率低下
- 内容单一，缺乏差异化
- 滞后于市场热点，更新慢

### 解决方案
- 构建批量生成视频脚本的智能 Agent 平台
- 支持多平台差异化脚本 (抖音/快手/微博等平台风格不同)
- 批量生成，大幅提升效率

### 技术价值主张
- **Agent 工程化**: LangGraph.js 状态编排实践
- **全栈能力**: NestJS 后端 + 前端交互完整闭环
- **业务理解**: 从真实痛点出发，而非技术演示
- **技术决策**: 每个关键选型都有清晰理由
