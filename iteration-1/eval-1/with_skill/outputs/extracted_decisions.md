# Claude Code Session 摘要 - Video Script Agent 项目

## 项目概述
- **项目名称**: Video Script Agent (视频脚本生成 Agent)
- **技术栈**: LangChain.js + BullMQ + NestJS + TypeScript + PostgreSQL
- **开发时间**: 2026年2月-3月
- **主要功能**: 基于用户输入的短视频主题，自动生成分镜脚本、对话文案和拍摄建议

## 关键决策对话片段

### 1. 架构选型讨论 (2026-02-15)
**用户**: 我想用 LangChain.js 构建一个视频脚本生成 Agent，需要处理并发请求和任务队列，有什么推荐架构？

**Claude Code**: 对于需要处理并发和任务队列的场景，建议采用：
1. **NestJS** 作为后端框架，提供依赖注入和模块化架构
2. **BullMQ** 作为任务队列，支持 Redis 持久化和优先级队列
3. **LangChain.js** 作为 LLM 调用层，支持多模型切换和工具调用链

**用户**: 为什么选 BullMQ 而不是 RabbitMQ 或 Kafka？

**Claude Code**: BullMQ 的优势：
- 与 Redis 集成简单，无需额外中间件
- 内置重试、延迟队列、优先级队列功能
- 更适合中小规模实时任务处理
- 社区活跃，文档完善

### 2. Agent 设计决策 (2026-02-18)
**用户**: 如何设计视频脚本生成的 Agent 工作流？

**Claude Code**: 建议采用多步骤 Agent 链：
1. **主题分析 Agent**: 解析用户输入，提取关键元素（风格、时长、目标受众）
2. **结构规划 Agent**: 生成脚本大纲（开场、发展、高潮、结尾）
3. **对话生成 Agent**: 为每个场景生成具体对话
4. **分镜建议 Agent**: 提供拍摄角度、运镜建议

**用户**: 需要为每个 Agent 单独配置 LLM 吗？

**Claude Code**: 不需要，可以使用同一个 LLM 实例，但需要：
- 为每个 Agent 设计专门的 system prompt
- 维护独立的对话历史
- 实现工具调用链，让 Agent 之间可以传递结构化数据

### 3. 状态管理讨论 (2026-02-22)
**用户**: 如何跟踪长时运行的脚本生成任务状态？

**Claude Code**: 建议设计三级状态管理：
1. **任务队列状态** (BullMQ): pending → active → completed/failed
2. **业务逻辑状态**: analyzing → planning → generating → reviewing
3. **用户可见状态**: 通过 WebSocket 实时推送进度

**用户**: 需要持久化中间结果吗？

**Claude Code**: 建议持久化：
- 每个 Agent 的输入输出，便于调试和重试
- 最终生成的脚本版本历史
- 用户反馈和评分数据，用于后续模型优化

### 4. 性能优化讨论 (2026-02-25)
**用户**: LLM 调用比较慢，如何优化用户体验？

**Claude Code**: 可以考虑以下策略：
1. **流式响应**: 使用 Server-Sent Events 或 WebSocket 逐步返回结果
2. **缓存层**: 对相似主题的脚本进行缓存（Redis + 语义相似度匹配）
3. **预生成模板**: 为常见主题准备模板，减少 LLM 调用次数
4. **并发处理**: 使用 BullMQ 的 worker 池并行处理不同步骤

**用户**: 缓存策略怎么设计？

**Claude Code**: 建议二级缓存：
- **内存缓存**: 高频主题，TTL 5分钟
- **Redis 缓存**: 所有生成结果，TTL 24小时
- **数据库持久化**: 用户保存的脚本，永久存储

### 5. 错误处理设计 (2026-02-28)
**用户**: LLM API 可能不稳定，如何设计容错机制？

**Claude Code**: 建议多层容错：
1. **重试机制**: 指数退避重试，最多3次
2. **降级方案**: LLM 失败时返回预定义模板
3. **超时控制**: 每个 Agent 步骤设置独立超时（分析30s，生成60s）
4. **健康检查**: 定期测试 LLM API 可用性

**用户**: 如何监控任务失败？

**Claude Code**: 建议实现：
- BullMQ 失败队列自动重试
- Sentry 集成错误上报
- 自定义监控面板，展示任务成功率、平均耗时

### 6. 前端架构讨论 (2026-03-02)
**用户**: 前端用什么技术栈比较好？

**Claude Code**: 考虑到字节跳动技术栈偏好，建议：
- **React 18+** 配合 TypeScript
- **TanStack Query** 处理数据获取和缓存
- **Zustand** 或 **Jotai** 状态管理
- **Tailwind CSS** 样式
- **Vite** 构建工具

**用户**: 需要实现实时进度展示吗？

**Claude Code**: 是的，建议：
- WebSocket 连接后端任务状态
- 进度条组件，显示当前处理步骤
- 允许用户中途取消任务

### 7. 部署方案讨论 (2026-03-05)
**用户**: 如何部署这个项目？

**Claude Code**: 建议容器化部署：
- **Docker** 打包应用
- **Docker Compose** 本地开发环境（NestJS + Redis + PostgreSQL）
- **Kubernetes** 生产环境部署
- **GitHub Actions** CI/CD 流水线

**用户**: 环境变量怎么管理？

**Claude Code**: 建议：
- `.env` 文件本地开发
- Kubernetes ConfigMap/Secret 生产环境
- 不同环境（dev/staging/prod）独立配置
