# 面试题整理 - SaaS 平台项目 (Turborepo + Next.js 14 + Prisma + tRPC + AI Agent)

## 项目概述
- **技术栈**: Turborepo + Next.js 14 + Prisma + tRPC + PostgreSQL + Redis + Yjs + AI Agent
- **架构特点**: Monorepo、多租户 SaaS、实时协作、AI Agent 集成
- **项目规模**: 全栈项目，包含前端、后端、共享类型、数据库、实时同步、AI Agent 等多个模块

## 一、Monorepo 与工程化架构

### 1.1 基础问题
1. **为什么选择 Turborepo 而不是 Lerna 或 Nx 来管理 monorepo？**
   - 考察点：monorepo 工具选型决策能力
   - 回答要点：对比 Turborepo、Lerna、Nx 的核心差异，说明选择理由（轻量、Vercel 生态集成、缓存机制）

2. **Turborepo 的远程缓存是如何工作的？有什么价值？**
   - 考察点：构建工具高级特性理解
   - 回答要点：解释文件哈希、任务输出缓存原理，说明对 CI/CD 和团队协作的价值

### 1.2 深度问题
3. **在 monorepo 中如何管理共享类型和工具函数？**
   - 考察点：类型安全架构设计
   - 回答要点：描述 packages/shared/ 包的设计，TypeScript 项目引用配置，tRPC 端到端类型安全

4. **如果要从 monorepo 迁移到微服务架构，会遇到哪些挑战？**
   - 考察点：架构演进思维
   - 回答要点：分析 monorepo 在微服务场景的局限性，提出渐进式迁移策略

## 二、多租户 SaaS 架构

9. **在多租户环境下，如何设计缓存策略？**
